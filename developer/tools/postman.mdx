## 下载集合

Waffo 提供官方 Postman 集合，涵盖所有核心 API 接口，包含预配置的请求示例和环境变量，帮助您快速上手 API 调试。

### 获取方式

**方式一：直接导入链接**

在 Postman 中点击「Import」→「Link」，输入以下地址：

```text
https://api.waffo.com/postman/collection.json
```

**方式二：下载文件**

访问 [Waffo 开发者中心](https://portal.waffo.com/developer/postman) 下载最新版本的 Postman Collection 文件（`.json` 格式），然后在 Postman 中导入。

**方式三：Fork 公开集合**

在 Postman Public API Network 搜索「Waffo Payment API」，直接 Fork 官方集合到您的工作空间。

## 使用说明

### 集合结构

Waffo Postman 集合按 API 模块组织，包含以下文件夹：

```text
Waffo Payment API
├── Authentication        # 认证测试
├── Payment Intents       # 支付意图
│   ├── Create Payment Intent
│   ├── Confirm Payment Intent
│   ├── Retrieve Payment Intent
│   └── Cancel Payment Intent
├── Checkout Sessions     # 收银台 Session
│   ├── Create Session
│   └── Retrieve Session
├── Refunds              # 退款
│   ├── Create Refund
│   └── Retrieve Refund
├── Subscriptions        # 订阅
│   ├── Create Subscription
│   ├── Update Subscription
│   └── Cancel Subscription
├── Payouts              # Payout 代发
│   ├── Create Payout
│   └── Retrieve Payout
└── Webhooks             # Webhook 测试
```

### 使用内置测试脚本

集合中每个请求都包含预置的测试脚本，运行后自动验证响应：

```javascript
// 示例：创建 PaymentIntent 的测试脚本
pm.test("Status code is 200", () => {
    pm.response.to.have.status(200);
});

pm.test("Response has client_secret", () => {
    const response = pm.response.json();
    pm.expect(response.client_secret).to.include('pi_');
});

// 将 PaymentIntent ID 保存为环境变量，供后续请求使用
pm.environment.set("payment_intent_id", pm.response.json().id);
```

### 运行集合测试

使用 Postman Collection Runner 批量运行所有测试：

1. 点击集合旁的「...」菜单 → 「Run collection」
2. 选择测试环境
3. 点击「Run Waffo Payment API」
4. 查看测试报告，确认所有接口正常响应

### 使用 Newman 在 CI/CD 中运行

```bash
# 安装 Newman（Postman 命令行工具）
npm install -g newman

# 运行集合
newman run waffo-api-collection.json \
  --environment waffo-test-env.json \
  --reporters cli,junit \
  --reporter-junit-export results.xml
```

## 常见问题

**Q: 导入后请求返回 401 错误？**

A: 请检查环境变量中的 `secret_key` 是否已正确填写，确认激活了正确的环境。

**Q: 如何测试 Webhook？**

A: 集合中包含 Webhook 触发器请求，配合 [Waffo CLI](/developer/tools/sdk) 可将事件转发到本地服务器进行测试。

**Q: 集合版本如何更新？**

A: 关注 Waffo 开发者中心的更新通知，或重新导入最新集合。已有的环境变量配置不会丢失。
