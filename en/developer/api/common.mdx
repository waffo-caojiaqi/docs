## Refund API

`POST /v1/refunds`

Initiate a refund for a successful payment. Supports full refunds and partial refunds.

**Request parameters:**

| Parameter | Type | Required | Description |
|------|------|------|------|
| payment_id | string | Yes | The payment ID to refund |
| amount | integer | No | Refund amount (if omitted, a full refund is issued) |
| reason | string | No | Refund reason (`duplicate`, `fraudulent`, `customer_request`) |
| metadata | object | No | Custom extension fields |

**Notes:**
- The same payment can be partially refunded multiple times, but the cumulative refunded amount must not exceed the original payment amount
- Refunds typically reach the user’s account within 5–10 business days, depending on the card issuer’s processing time
- No additional fees are charged for refunds, but the fees from the original payment are not returned

## Query APIs

**Common list query parameters:**

All list APIs support the following pagination parameters:

<ParamField name="limit" type="integer">
  Items per page, range 1–100, default 20
</ParamField>

<ParamField name="starting_after" type="string">
  Cursor pagination: return records after this ID
</ParamField>

<ParamField name="ending_before" type="string">
  Cursor pagination: return records before this ID
</ParamField>

<ParamField name="created_after" type="integer">
  Created time lower bound (Unix timestamp)
</ParamField>

<ParamField name="created_before" type="integer">
  Created time upper bound (Unix timestamp)
</ParamField>


**Response format:**

```json
{
  "object": "list",
  "data": [...],
  "has_more": true,
  "total_count": 156
}
```

## Webhook

Waffo uses Webhooks to push payment events to your server in real time.

**Webhook signature verification:**

Each Webhook request header includes the `Waffo-Signature` field, which is used to verify the request source.

```javascript
const crypto = require('crypto');

function verifyWebhookSignature(payload, signature, secret) {
  const expectedSig = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSig)
  );
}
```

**Main Webhook event list:**

| Event | Description |
|------|------|
| payment.succeeded | Payment succeeded |
| payment.failed | Payment failed |
| payment.refunded | Refund completed |
| subscription.created | Subscription created |
| invoice.payment_succeeded | Subscription renewal succeeded |
| invoice.payment_failed | Subscription renewal failed |
| payout.succeeded | Payout settled |
| dispute.created | New dispute/chargeback |

**Webhook retry mechanism:**

If the Webhook endpoint returns a non-2xx status code, Waffo will retry using an exponential backoff strategy, up to 5 retries, at intervals of 5 minutes, 30 minutes, 2 hours, 6 hours, and 24 hours.

**Error code reference:**

The Waffo API uses standard HTTP status codes. Error response format:

```json
{
  "error": {
    "code": "card_declined",
    "message": "Your card was declined.",
    "decline_code": "insufficient_funds",
    "param": null
  }
}
```

For the full list of error codes, use the [Error Code Lookup Tool](/developer/tools/overview) in the Merchant Portal.
