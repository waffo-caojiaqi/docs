## Chinese version

### 1. Product introduction

#### 1. Overview

Waffo Checkout is Waffo’s standardized checkout product designed to centrally manage the interaction flow for global payments. Merchants can offer users a secure, smooth, and consistent payment experience across multiple modes such as one-time payments and subscription payments.

#### 2. Core capabilities

1. **Unified payment mode**: A single integration supports both one-time purchases (One-time) and subscription billing (Subscription).
2. **Global adaptation**: Intelligently matches local languages and payment methods to eliminate friction in cross-border payment experiences.
3. **Multi-device responsiveness**: Whether desktop Web or mobile H5, the UI is deeply optimized to improve conversion rates.
4. **Highly customizable**: Provides rich configuration options such as brand color, text color, and border radius to blend perfectly with your brand style.

---

### 2. Two Checkout experiences

Waffo provides two primary integration approaches, both fully supporting all payment methods and subscription features.

#### 1. Hosted Checkout

**What is Hosted Checkout?**

Hosted Checkout is a fully Waffo-hosted checkout page. Merchants only need to call the API to create a payment session, obtain a secure Waffo-generated link, and then redirect users to that link. Users complete the entire payment flow under Waffo’s domain and return to the merchant website after payment is completed.

**Key features:**

- **Zero frontend development**: Merchants do not need to build any frontend payment pages. Page rendering, form validation, error handling, 3DS authentication, and more are all handled by Waffo. With just a single backend API call, you get a complete payment capability.
- **Continuous auto-optimization**: Waffo continuously optimizes checkout layout, copy, and interaction logic to improve conversion rates. All optimizations are applied automatically without additional development.
- **Worry-free security and compliance**: Merchants do not need to handle sensitive card data, minimizing PCI DSS compliance effort (SAQ A level). Waffo handles secure processing and storage of all payment data.
- **Zero maintenance cost**: Page updates, new payment method additions, and security patch upgrades are all handled by the Waffo team.

#### 2. Embedded checkout (Embedded Checkout SDK)

Embed the Waffo checkout component directly into the merchant’s own checkout page to provide a seamless payment experience.

**Key features:**

- **Brand consistency**: The payment interface is integrated into the merchant website. Users do not need to leave the page, maintaining visual consistency.
- **Deep customization**: Customize theme colors, fonts, border radius, and background colors to fully match merchant brand guidelines.
- **Higher conversion rate**: Users stay on the merchant page, reducing drop-off from redirects and improving checkout conversion.
- **Flexible layout**: Embed the checkout into a sidebar, modal, full-screen view, or other layout forms.

### FAQ

**Q: What is the PCI compliance level difference between Hosted Checkout and Embedded SDK?**

A: Hosted Checkout meets SAQ A (the lowest compliance requirements), and merchants do not touch card data at all. Embedded SDK meets SAQ A-EP, and Waffo.js ensures card data is encrypted before it leaves the browser. Neither requires the merchant server to handle raw card numbers.

**Q: Does Checkout support remembering user payment methods?**

A: Yes. Through Tokenization, after the user’s first payment they can choose to save the payment method. On the next checkout, they can select a saved card directly without re-entering it.
