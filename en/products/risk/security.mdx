## Data and payment security

Waffo builds its entire payment system on a security-first foundation. From data transmission to storage, and from transaction verification to identity authentication, every step adopts the highest industry-standard security technologies to ensure the security of merchants’ and users’ funds and data.

### Transport security

- **TLS 1.2 / 1.3**: All API communications enforce TLS-encrypted transport to prevent man-in-the-middle attacks
- **Certificate pinning**: The mobile SDK implements certificate pinning to prevent certificate forgery
- **HSTS**: HTTP Strict Transport Security, which forces browsers to use HTTPS connections

### Data-at-rest security

- **AES-256 encryption**: Sensitive data at rest (card numbers, bank account information) is encrypted using the AES-256 algorithm
- **Tokenization**: Primary account numbers never enter merchant systems; Waffo generates irreversible tokens as storage substitutes
- **Key management**: Encryption keys are managed using HSMs (Hardware Security Modules), rotated regularly, and accessed under least-privilege controls
- **Data isolation**: Merchant data is strictly isolated; the multi-tenant architecture ensures no data cross-contamination

### Real-time fraud detection

Waffo includes a machine learning-driven, real-time Risk Control engine that performs multi-dimensional risk assessments on every transaction before completion:

| Detection dimension | Description |
|----------|------|
| Device fingerprinting | Identifies device characteristics and detects abnormal or cloned devices |
| Behavioral analytics | Analyzes user interaction patterns to identify bots |
| Geolocation | Detects anomalous geo-hopping (e.g., logins from different countries in a short period) |
| Velocity rules | Identifies high-frequency transactions in a short period (card testing attacks) |
| Blacklists | Real-time blocking of known fraudulent emails, IPs, and devices |
| Network graph | Uses correlation analysis to identify organized fraud rings |

## Certifications such as PCI and GDPR

### PCI DSS Level 1

Waffo has achieved **PCI DSS Level 1** certification—the highest level of the Payment Card Industry Data Security Standard, applicable to institutions processing more than 6 million transactions annually.

**What this certification means**:
- Merchants can transmit payment data to Waffo for processing without maintaining PCI DSS compliance themselves
- Merchants only need to complete **SAQ A** (the simplest level) of self-assessment, significantly reducing compliance costs
- Annual independent audits are performed by a QSA (Qualified Security Assessor)

### GDPR compliance

Waffo strictly complies with the EU General Data Protection Regulation (GDPR):

- **Data Processing Agreement (DPA)**: As a data processor, Waffo signs a DPA with merchants
- **Data subject rights**: Supports requests for access, correction, deletion (right to be forgotten), and more
- **Data localization**: EU user data is stored in data centers within the EU
- **Privacy by Design**: Minimizes data collection and applies the highest privacy protection by default

### Other certifications

| Certification | Description |
|------|------|
| SOC 2 Type II | Third-party audit of security, availability, and data integrity |
| ISO 27001 | International standard for information security management systems |
| 3DS 2.x certification | EMVCo 3-D Secure protocol certification |

## Internal compliance / Risk Control mechanisms

### AML / KYC compliance

Waffo implements strict Anti-Money Laundering (AML) and Know Your Customer (KYC) procedures:

- **Merchant due diligence (KYB)**: Before onboarding, merchants complete business background checks, including verification of business licenses and beneficial owners
- **Transaction monitoring**: Real-time monitoring of all transactions, with automatic manual review triggered for transactions exceeding compliance thresholds
- **Suspicious Activity Report (SAR)**: Reports are submitted to regulators when suspected money laundering activity is identified
- **Sanctions screening**: Real-time screening of all parties against sanctions lists such as OFAC, EU, and UN

### Risk Control rules engine

Merchants can configure custom Risk Control rules via the Waffo Dashboard:

```text
IF 单笔金额 > 10,000 USD
AND 用户账龄 < 30 天
THEN 触发 3DS 验证

IF 24小时内同一卡号交易次数 > 5
THEN 阻断交易并发送警报
```

### Security Operations Center (SOC)

Waffo’s SOC operates 24/7 to monitor and respond to platform security incidents in real time:

- Mean time to detect (MTTD): < 5 minutes
- Mean time to respond (MTTR): < 30 minutes
- Regular penetration testing: Conducted quarterly by third-party security firms

## User data and privacy protection policy

### Data collection principles

Waffo follows the principle of data minimization and collects only the data necessary to deliver payment services:

| Data type | Purpose | Retention period |
|----------|----------|----------|
| Payment card information | Transaction processing (tokenized storage) | No more than 5 years |
| Transaction records | Reconciliation, dispute handling, compliance | 7 years (regulatory requirement) |
| Device information | Fraud detection | 13 months |
| IP address | Risk assessment | 6 months |

### Data sharing

Waffo shares user data only under the following circumstances:
- Partner institutions necessary to complete payment processing (banks, card networks)
- Information disclosure to regulators as required by law
- Third-party services explicitly authorized in writing by the merchant

**Waffo commitment**: We do not sell user data to third parties and do not use it for ad targeting.
