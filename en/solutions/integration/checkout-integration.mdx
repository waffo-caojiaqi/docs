## Integration steps

Hosted checkout integration takes only three steps:

**Step 1: Create a payment session**

On your server, call the Waffo API to create a Checkout Session, specifying the payment amount, currency, product information, and callback URLs:

```bash
POST https://api.waffo.com/v1/checkout/sessions
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{
  "amount": 9900,
  "currency": "USD",
  "order_id": "your-order-id-001",
  "success_url": "https://yoursite.com/payment/success",
  "cancel_url": "https://yoursite.com/payment/cancel",
  "webhook_url": "https://yoursite.com/webhooks/waffo"
}
```

**Step 2: Redirect to checkout**

Redirect the user to the `checkout_url` returned in the response. The user completes the payment on the Waffo-hosted page.

**Step 3: Handle the payment result**

- After the user completes the payment, they are redirected back to `success_url` (the URL includes `session_id`)
- At the same time, Waffo sends a `payment.succeeded` event to `webhook_url`
- **Important**: Use the webhook callback as the source of truth to update order status, and do not rely only on the page redirect

## Parameter configuration

Key parameters supported when creating a Checkout Session:

<ParamField name="amount" type="integer" required>
  Payment amount, in the smallest currency unit (e.g., cents)
</ParamField>

<ParamField name="currency" type="string" required>
  3-letter ISO currency code, such as USD, EUR
</ParamField>

<ParamField name="order_id" type="string" required>
  Merchant-side order ID; must be unique
</ParamField>

<ParamField name="success_url" type="string" required>
  Redirect URL after a successful payment
</ParamField>

<ParamField name="cancel_url" type="string" required>
  Redirect URL after the user cancels the payment
</ParamField>

<ParamField name="webhook_url" type="string">
  Payment result callback URL (recommended)
</ParamField>

<ParamField name="customer_email" type="string">
  Customer email, used to send a payment receipt
</ParamField>

<ParamField name="locale" type="string">
  Checkout display language, such as zh-CN, en, ja
</ParamField>

<ParamField name="payment_methods" type="array">
  List of payment methods to display (restricted)
</ParamField>

<ParamField name="metadata" type="object">
  Custom key-value pairs, returned as-is
</ParamField>
