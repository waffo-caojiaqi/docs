Waffo Docs provides a set of reusable React components that go beyond Mintlify's built-ins.
Use these when you need to match specific design patterns from the product design system.

## When to use custom vs built-in components

| Situation | Use |
|---|---|
| Simple callout, note, or warning | `<Note>`, `<Warning>` (Mintlify built-in) |
| Sequential steps | `<Steps>` (Mintlify built-in) |
| Feature cards with branded icon style | `CapabilityCard` (custom) |
| Payment method breakdown with logos | `PaymentMethodPanel` (custom) |
| Side-by-side model/plan comparison | `BusinessModelCard` (custom) |
| Horizontal flow diagram with alert | `BillingFlowStepper` (custom) |

## CapabilityCard

A feature highlight card with an icon, title, and description. Use `CapabilityGrid` to arrange them in a 2-column layout.

```mdx
import { CapabilityCard, CapabilityGrid } from '/snippets/components/capability-card.jsx'

<CapabilityGrid>
  <CapabilityCard
    icon="🗓"
    title="Flexible Billing Cycles"
    description="Supports weekly, monthly, quarterly, yearly and custom intervals."
  />
  <CapabilityCard
    icon="⚙️"
    title="Smart Retry Mechanism"
    description="Built-in retry strategies that optimize success rates."
  />
</CapabilityGrid>
```

**Live example:**

  />
</CapabilityGrid>

## PaymentMethodPanel

A card showing a payment method category with supported logos and a feature checklist.

```mdx
import { PaymentMethodPanel } from '/snippets/components/payment-method-panel.jsx'

<PaymentMethodPanel
  title="Bank Cards (Credit/Debit Cards)"
  logos={["JCB", "Visa", "Mastercard"]}
  features={[
    {
      title: "Tokenization Security",
      description: "A Token is created on first payment; subsequent deductions use MIT mode.",
    },
    {
      title: "Auto-Update",
      description: "Supports VAU and ABU for automatic card updates.",
    },
  ]}
/>
```

## BusinessModelCard

Comparison card for side-by-side plan or model comparisons (A/B layout).

```mdx
import { BusinessModelCard, BusinessModelGrid } from '/snippets/components/business-model-card.jsx'

<BusinessModelGrid>
  <BusinessModelCard
    label="A"
    title="Service-First"
    tagline="Prioritize service continuity; tolerate payment latency."
    mechanism="Service continues even if deduction fails. Concurrent billing cycles are allowed."
    bestFor="Video streaming, SaaS software, user retention–focused services."
  />
  <BusinessModelCard
    label="B"
    title="Payment-First"
    tagline="Strictly pay before service; zero tolerance for arrears."
    mechanism="Service suspends immediately on failure until payment succeeds."
    bestFor="Physical goods, game items, cost-sensitive businesses."
  />
</BusinessModelGrid>
```

## BillingFlowStepper

Horizontal step-by-step flow diagram, with an optional failure/warning alert box.

```mdx
import { BillingFlowStepper } from '/snippets/components/billing-flow-stepper.jsx'

<BillingFlowStepper
  title="Billing Flow"
  steps={["Expiry Detection", "Generate MIT Order", "Initiate Deduction", "Success / Failure Retry"]}
  alert={{
    title: "Failure Handling",
    lines: [
      "Default strategy retries 24 hours after failure, and again 48 hours later.",
      "Supports automatic failure notification emails to guide users to update their payment method.",
    ],
  }}
/>
```

## Adding a new component

1. Create `snippets/components/<your-component>.jsx`.
2. Export the component (and an optional grid/wrapper).
3. Add JSDoc at the top with a usage example.
4. Add it to the table at the top of this page.
5. If you have a design file, tell Claude Code: "build a custom component from this design" — it will follow the Waffo design tokens automatically.
