## 中文版

### 一、产品介绍

#### 1. 产品概览（Overview）

为单笔交易提供清晰、稳定的支付能力。通过统一的 API 接口，支持全球多种支付方式，帮助商户高效完成每一笔交易。

#### 2. 什么是一次性支付？

一次性支付（One-time Payment）是指针对单笔交易进行一次收费的支付模式。在这种模式下，每一笔支付均为独立交易，不涉及周期性扣款或自动续费。由于其结构清晰、流程直接，它是最基础、也是最常见的支付形态之一，适用于广泛的商业场景。

#### 3. 核心能力

1. **统一接入**：一套接口集成信用卡、电子钱包等多种支付方式，降低系统复杂度。
2. **清晰反馈**：支付全链路状态可视，关键节点实时反馈，便于商户处理与追踪。
3. **集中管理**：统一的交易管理后台（Dashboard），支持商户进行对账、查询与报表导出。
4. **灵活收银台集成**：统一 API 接入，商户可自由选择 Hosted 托管页面或 Embedded 内嵌组件，灵活适配业务场景。

### 常见问题 (FAQ)

**Q: 一次性支付和订阅支付有什么区别？**

A: 一次性支付每笔交易独立，不存储持卡人信息用于后续扣款。订阅支付会通过 Tokenization 保存支付凭证，用于后续周期性自动扣款。

**Q: 支付失败后如何处理？**

A: Waffo 会通过 Webhook 推送失败原因（包含标准化错误码）。商户可根据错误类型决定是否引导用户重试或更换支付方式。

### 5. FAQ (Frequently Asked Questions)

**Q: What's the difference between one-time payment and subscription payment?**

A: One-time payments are independent per transaction with no stored credentials. Subscription payments use Tokenization to save payment credentials for subsequent recurring charges.

**Q: How are payment failures handled?**

A: Waffo sends failure reasons (including standardized error codes) via Webhook. Merchants can decide whether to guide users to retry or switch payment methods based on the error type.
