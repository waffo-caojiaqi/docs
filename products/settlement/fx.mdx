## 服务介绍

Waffo 外汇服务（FX）帮助跨境商户轻松处理多货币收款和结算。商户可以用用户习惯的本地货币收款，Waffo 负责完成货币转换，并以商户指定的结算货币划付资金，免去复杂的多货币账户管理负担。

Waffo 与主要国际银行和外汇流动性提供商合作，提供有竞争力的汇率和快速的换汇执行能力。

## 外汇管理

### 多货币收款

商户可以一次性配置多种收款货币（Payment Currency），Waffo 自动检测用户所在地区并展示对应货币的价格，提升本地化购物体验。

**工作流程**：
1. 商户在控制台配置支持的收款货币（如 USD、EUR、JPY、SGD）
2. 用户在商户页面看到本地货币标价
3. 用户完成支付，资金以其支付货币进入 Waffo
4. Waffo 按结算时汇率自动换算为商户结算货币
5. 商户收到统一结算货币的款项

### 汇率机制

Waffo 提供两种汇率模式：

#### 实时汇率（Spot Rate）
- 换汇时刻的实时市场汇率，加 Waffo 服务价差
- 适合不需要精确汇率预算的商户
- 汇率波动风险由商户承担

#### 锁汇（Forward Lock）
- 商户在交易前锁定目标汇率，有效期内汇率固定
- 适合对成本精确性要求高的商户（如定价已锁定的产品）
- 需提前申请，最长锁汇周期：90 天

### 汇率查询

商户可通过 API 实时查询最新汇率：

```bash
# 查询汇率
GET /v1/fx/rates?from=USD&to=CNY

# 响应示例
{
  "from": "USD",
  "to": "CNY",
  "rate": 7.2456,
  "timestamp": 1700000000,
  "valid_until": 1700003600
}
```

### 多货币结算账户

商户可配置多个结算账户，分别接收不同货币：

```text
USD 收款 → 美元账户（免换汇）
EUR 收款 → 欧元账户（免换汇）
其他货币收款 → 换汇后统一划入指定账户
```

## 外汇敞口管理

### 什么是外汇敞口

外汇敞口（FX Exposure）是指商户因持有多种货币资产或负债而面临的汇率波动风险。例如：商户以 EUR 收款但以 USD 支付成本，若 EUR/USD 汇率下跌，则实际收益缩水。

### 敞口管理策略

#### 实时换汇（Immediate Conversion）
每笔交易完成后立即以实时汇率换算为结算货币，消除汇率敞口。

- **优点**：无汇率风险，收益可预测
- **缺点**：若在汇率低点换汇，可能损失潜在收益

#### 延迟换汇（Batch Conversion）
将同一货币的收款汇总后批量换汇，可选择最优时机。

- **优点**：可在汇率较高时集中换汇，优化收益
- **缺点**：存在汇率下跌风险，需要主动管理

#### 自然对冲（Natural Hedging）
将同一货币的收款与该货币的支出（如供应商付款、员工薪资）相互抵销。

- **适用场景**：在目标国家同时有收款和支出的商户

### 敞口报告

Waffo 控制台提供实时的外汇敞口仪表盘：

| 指标 | 说明 |
|------|------|
| 当前敞口金额 | 尚未换汇的各币种余额及等值 USD |
| 汇率变动影响 | 基于当前市场价，敞口资产的浮动盈亏 |
| 历史换汇记录 | 每次换汇操作的详细记录和汇率 |
| 汇率预警设置 | 当特定汇率达到预设阈值时发送通知 |

<Warning>
外汇市场波动可能对商户结算收入产生影响。建议商户根据业务规模和风险承受能力选择合适的外汇管理策略，必要时咨询专业财务顾问。
</Warning>
