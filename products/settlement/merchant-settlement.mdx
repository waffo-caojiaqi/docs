## 服务介绍

Waffo 商户结算服务负责将商户收款资金按约定周期结算至商户指定银行账户。Waffo 提供透明的资金流水记录、灵活的结算周期配置，以及完整的对账报表，帮助商户高效完成财务管理工作。

结算资金 = 交易收款总额 - 平台手续费 - 退款金额 - 争议扣款 - 保证金冻结

## 商户结算和提现

### 结算周期

Waffo 支持以下结算频率，商户可与客户成功经理协商选择：

| 结算周期 | 说明 | 适用商户类型 |
|----------|------|-------------|
| T+1 日结 | 交易次日结算至商户账户 | 大型成熟商户，资金需求紧迫 |
| T+3 滚动结算 | 每笔交易延迟 3 天结算 | 标准商户 |
| 周结算 | 每周一次，结算前一周资金 | 中小型商户 |
| 月结算 | 每月固定日期（1日/15日）结算 | 低频交易商户 |

<Info>
T+1 日结需要商户满足一定的交易量门槛和信用评分。新商户默认适用 T+3 滚动结算周期。
</Info>

### 结算流程

<Steps>
  <Step title="交易完成">
    用户完成支付，资金进入 Waffo 托管账户，交易状态更新为 `succeeded`。
  </Step>
  <Step title="结算计算">
    Waffo 在结算日统计周期内的净收款金额，扣除手续费、退款及保证金后，生成结算单。
  </Step>
  <Step title="结算审核">
    风控系统自动审核结算单，对异常交易进行人工复核。正常结算单将在 24 小时内完成审核。
  </Step>
  <Step title="资金划付">
    审核通过后，Waffo 通过银行转账将结算资金划付至商户绑定的结算账户。
  </Step>
  <Step title="结算通知">
    结算完成后，系统自动发送结算通知邮件，包含本次结算明细和流水对账文件。
  </Step>
</Steps>

### 提现管理

商户可在 Waffo 控制台发起手动提现，将可用余额提现至绑定的银行账户。

**提现限制**：
- 单次提现最低金额：100 USD（或等值货币）
- 提现到账时效：T+1 ~ T+3 个工作日
- 每日提现次数限制：根据商户等级不同而异

**提现状态**：
- `pending`：申请已提交，等待处理
- `in_transit`：资金已划出，银行处理中
- `paid`：资金已到达商户账户
- `failed`：提现失败，资金退回可用余额

## 商户保证金

### 保证金制度说明

保证金（Security Deposit / Reserve）是 Waffo 为控制商户信用风险而冻结的资金。保证金从每次结算中按比例扣留，不影响商户正常运营，商户合同到期后或达到释放条件时全额退还。

### 保证金类型

| 类型 | 说明 |
|------|------|
| 滚动保证金（Rolling Reserve） | 每笔结算中扣留固定比例（通常 5%~10%），滚动保留 90~180 天后释放 |
| 固定保证金（Fixed Reserve） | 一次性冻结固定金额，适用于高风险商户或新商户 |
| 增强保证金 | 当商户拒付率超过阈值时临时增加的保证金比例 |

### 保证金释放条件

- 商户拒付率连续 3 个月低于 1%
- 商户交易量和历史合规记录达到优质等级
- 合同约定的保留期限届满

## 商户报表和对账

### 报表类型

| 报表 | 更新频率 | 内容 |
|------|----------|------|
| 交易明细报表 | 实时 | 每笔交易的完整信息 |
| 结算报表 | 每次结算后 | 结算金额、手续费、退款明细 |
| 对账报表 | 每日/每周/每月 | 与商户系统的资金核对数据 |
| 保证金报表 | 每月 | 保证金变动、余额、释放计划 |
| 纠纷报表 | 实时 | 争议、拒付案件追踪 |

### 对账文件格式

Waffo 支持以下对账文件格式：

- **CSV**：通用格式，适合 Excel 处理
- **XLSX**：Excel 原生格式，支持多 Sheet
- **JSON**：通过 API 拉取，适合系统自动对账
- **MT940**：银行标准格式，适合 ERP 系统集成

### API 对账接口

```bash
# 拉取结算报表
GET /v1/reports/settlement?
  settlement_id=stl_xxx&
  format=json

# 拉取交易明细
GET /v1/transactions?
  created_gte=1700000000&
  created_lte=1700086400&
  status=succeeded&
  limit=100
```

<Tip>
建议商户配置 `settlement.paid` Webhook 事件，在结算完成时自动触发内部对账流程，确保财务数据实时同步。
</Tip>
