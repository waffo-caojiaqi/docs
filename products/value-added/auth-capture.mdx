## 预授权

预授权（Authorization）是一种先冻结用户资金、后实际扣款的支付模式。商户在用户下单时发起预授权，银行在用户账户中冻结对应金额，但资金尚未实际转移。待商户确认发货或服务完成后，再发起请款（Capture）完成实际扣款。

预授权适用于以下场景：

- **酒店/租车预订**：预订时冻结资金，退房/还车时按实际费用请款，多余冻结自动释放
- **分步发货**：大额订单分批发货，按实际发货金额分次请款
- **延迟确认订单**：下单时冻结，人工审核或配货后再扣款
- **超额授权**：冻结略高于预估金额（如酒店额外消费），最终按实际金额请款

预授权的有效期通常为 7-30 天（因卡组织和发卡行规则不同），超期未请款的预授权将自动失效。

## 请款

请款（Capture）是对已完成预授权的交易发起实际资金划转的操作：

- **全额请款**：对预授权金额全额扣款，最常见的请款方式
- **部分请款**：只扣取预授权金额的一部分，剩余冻结金额自动释放（如实际消费低于预估）
- **多次请款**：对同一预授权发起多次部分请款，总额不超过预授权金额（如分批发货场景）
- **取消授权（Void）**：在请款前取消预授权，立即释放冻结资金

预授权和请款操作均可通过 Waffo API 或商户 Portal 发起。详细接口参数请参阅[一次性支付 API](/developer/api/one-time-payment) 文档中的 Auth-Capture 部分。
