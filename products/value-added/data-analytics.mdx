## 运营日会成功率分析

Waffo 数据分析平台为商户提供实时的支付成功率监控，是支付运营日会的核心数据来源。通过清晰的漏斗模型和趋势图表，帮助商户团队快速定位成功率波动原因。

### 成功率指标定义

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 授权成功率 | 授权成功笔数 / 授权请求总笔数 | 银行侧授权通过率 |
| 最终支付成功率 | 最终成功笔数 / 发起支付总笔数 | 用户完整完成支付的比例 |
| 3DS 通过率 | 3DS 验证通过笔数 / 3DS 触发笔数 | 强验证场景的通过率 |
| 充值完成率 | 用户进入收银台后最终完成支付的比例 | 结合前端行为数据分析 |

### 成功率分析维度

商户可按以下维度下钻分析成功率：

- **时间维度**：小时、天、周、月对比
- **支付方式**：各卡组织、数字钱包、本地支付方式分别统计
- **用户地区**：按国家/地区拆分，识别特定地区问题
- **失败原因码**：按发卡行返回的拒绝原因分类统计
- **商户应用**：多店铺商户支持按站点/应用拆分

### 实时交易流水

OTP 看板提供最近 100 笔交易的实时流水，展示：
- 交易时间、金额、货币
- 支付方式、卡 BIN 前六位
- 交易状态及发卡行原因码
- 3DS 验证状态

## 订阅看板

订阅看板（Subscription Dashboard）提供订阅业务的全周期健康度监控。

### 订阅核心指标

| 指标 | 说明 |
|------|------|
| MRR（月度经常性收入） | 当月活跃订阅的月费收入合计 |
| ARR（年度经常性收入） | MRR × 12 的年化收入预测 |
| 活跃订阅数 | 当前处于 `active` 状态的订阅总数 |
| 新增订阅 | 本期新注册的订阅数量 |
| 取消订阅 | 本期主动取消的订阅数量 |
| 续费失败率 | 自动续费扣款失败比例（MIT 失败率） |
| 用户流失率（Churn Rate） | 月度订阅用户流失比例 |

### 智能重试分析

订阅看板专门提供续费失败的重试效果分析：

- 各失败原因的重试成功率
- 最优重试时间窗口分析
- 重试挽回的月度收入贡献

## 错误码分析

错误码分析帮助商户了解支付失败的根本原因，并制定针对性的优化策略。

### 错误码分类

| 错误类型 | 说明 | 处理建议 |
|----------|------|----------|
| `insufficient_funds` | 持卡人余额不足 | 引导更换支付方式或分期 |
| `card_declined` | 发卡行拒绝（通用） | 建议用户联系发卡行 |
| `authentication_failed` | 3DS 验证失败 | 检查 3DS 集成是否正确 |
| `expired_card` | 卡片已过期 | 引导用户更新卡片信息 |
| `lost_card` | 卡片挂失 | 自动标记风险用户 |
| `processing_error` | 支付网关处理错误 | 可安全重试 |
| `do_not_honor` | 发卡行通用拒绝 | 建议用户联系发卡行 |

### 错误码趋势监控

商户可在错误码分析页面查看：
- 各错误码的出现频率和趋势
- 不同支付方式下的错误分布
- 与行业基准的对比（Waffo 聚合数据匿名对比）
- 错误码突增告警配置

## 收银台成功率分析

收银台成功率分析结合前端埋点数据，提供完整的支付漏斗可视化。

### 支付漏斗模型

```
展示收银台
    ↓ 用户选择支付方式
    ↓ 用户填写支付信息
    ↓ 用户点击确认支付
    ↓ 支付处理中（含 3DS）
    ↓ 支付成功
```

每个漏斗节点均统计转化率，帮助识别用户流失发生在哪个环节。

### 收银台 A/B 测试

Waffo 支持对收银台配置进行 A/B 测试，对比不同：
- 支付方式排列顺序
- UI 主题（深色/浅色）
- CTA 按钮文案

## 支付报告

Waffo 提供多种格式的定期支付报告，支持自动发送至指定邮箱。

| 报告类型 | 发送频率 | 内容摘要 |
|----------|----------|----------|
| 日报 | 每日早 9:00 | 昨日交易摘要、成功率变化 |
| 周报 | 每周一 | 上周整体业绩、对比分析 |
| 月报 | 每月 1 日 | 月度完整财务报告 |
| 自定义报告 | 按需 | 商户指定维度和时间范围 |

报告可导出为 **PDF**、**Excel**、**CSV** 格式，支持直接接入 BI 系统（如 Tableau、Power BI）。
